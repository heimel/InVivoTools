## Graph functions

Graphs of grouped experimental data can be made through the GUI by calling:

```matlab
graph_db
```

When the **Compute** button is clicked, the GUI-specific function
`graphdb_compute` is called. This calls:

```matlab
[record,hgraph] = compute_graphrecord(record)
```

`compute_graphrecord` then calls:

```matlab
groupgraph(groups,measures,varargin)
```

`groupgraph` interprets the graph record. It collects the groups, measures, and
extra arguments from the record and makes one call to:

```matlab
results = get_compound_measurements(groups,measures,varargin)
```

This retrieves the data for all requested measures and groups. After calling
`get_compound_measurements`, `groupgraph` formats the group and measure labels
and calls:

```matlab
h = ivt_graph(y,x,varargin)
```

`ivt_graph` is a general function for producing many types of figures from data
and accepts many options. See `help ivt_graph` for more information.

### Retrieving measurements

The function `get_compound_measurements` calls itself for each requested
measure. For each compound measure, such as
`10*given(tp:present)*tp:bouton`, it splits the expression into components and
performs the requested operation on the values. To retrieve individual
non-compound measures, it calls:

```matlab
results = get_measurements(groups,measure,varargin)
```

`get_measurements` loops over the requested groups and calls:

```matlab
results{g} = get_measurements_for_group(groupss(g),measuress,...);
```

If `value_per` is set to `'group'`, values are averaged per group:

```matlab
results{g} = nanmean(results{g},ndims(results{g}));
```

`get_measurements_for_group` loops over the mice in each group and calls:

```matlab
res = get_measurements_for_mouse(mouse,measure,...);
```

If `value_per` is set to `'mouse'`, values are averaged per mouse:

```matlab
res = nanmean(res,ndims(res));
```

`get_measurements_for_mouse` loops over the tests or stacks for each mouse and
calls:

```matlab
res = get_measurements_for_test(testrecord,mouse,measure,...);
```

Depending on `value_per`, it can take the mean or sum per test or stack:

```matlab
switch value_per
    case {'test','stack'}
        res = nanmean(res);
    case {'testsum','stacksum'}
        res = mynansum(res);
end
```

For each test record, values are retrieved with:

```matlab
val = get_compound_measure_from_record(record,measure,criteria,...)
```

If `value_per` is set to `'neurite'`, values are averaged per neurite at this
point.

The function `get_compound_measure_from_record` splits the measure further if
it still contains computations and calls:

```matlab
val = get_measure_from_record(record,measure,criteria,extra_options)
```

This returns the values for all cells or ROIs for a specific measure from
`record.measures` when the record matches `criteria`. Optical-imaging records
store results differently, so `get_measure_from_record` adapts them by calling:

```matlab
val = get_valrecord(record,measure)
```
